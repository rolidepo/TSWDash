# TSWDash

TSWDash is a Python + Flask dashboard for Train Sim World External Interface data.
It provides:

- Live MFD and touchscreen dashboards
- JSON-driven pages/components
- Route editor for service timetable files
- Timetable editor for schedule creation and timing helpers
- Support tools and schedule export endpoints

## What Is In This Project

- `src/main.py`: application entrypoint, starts poller and web server
- `src/api_poller.py`: polls TSW external API and stores runtime values
- `src/web_server.py`: Flask routes for UI pages and editor APIs
- `src/config.py`: API host/port/key loading and validation
- `src/templates/`: UI templates and JS component renderers
- `src/pages/`: page configuration JSON files
- `src/timetable/route/`: service timetable route JSON files
- `src/timetable/schedule/`: schedule JSON files

## Requirements

- Windows (recommended for TSW setup)
- Python 3.10+
- Train Sim World launched with HTTP API enabled (`-HTTPAPI`)
- API key file generated by game launch:
   - `Documents\My Games\TrainSimWorld6\Saved\Config\CommAPIKey.txt`

## Install

From the repository root:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Run

```powershell
cd src
python main.py
```

When started successfully, the app serves on port `5000`.

## Main URLs

- Dashboard root: `http://localhost:5000/`
- MFD view: `http://localhost:5000/mfd`
- Touchscreen view: `http://localhost:5000/touchscreen`
- Support tool: `http://localhost:5000/support-tool`
- Route editor: `http://localhost:5000/route-editor`
- Timetable editor: `http://localhost:5000/timetable-editor`
- Health endpoint: `http://localhost:5000/health`

## Key Configuration Files

- `src/api_endpoints.json`
   - API subscriptions/single calls and variable mapping
- `src/pages_config.json`
   - Active page setup and page selection metadata
- `src/dashboard_config.json`
   - Dashboard behavior/runtime display settings
- `src/runtime_variables.json`
   - Runtime variable persistence file used by the app

## Timetable Tooling Notes

- Route editor APIs are exposed under `/api/route-editor/*`
- Timetable editor APIs are exposed under `/api/timetable-editor/*`
- Timetable files are stored in:
   - `src/timetable/route/` (service route data)
   - `src/timetable/schedule/` (service schedules)

## Troubleshooting

If startup fails:

1. Verify TSW was launched with `-HTTPAPI`.
2. Confirm `CommAPIKey.txt` exists in the expected Documents folder.
3. Confirm dependencies are installed from `requirements.txt`.
4. Ensure port `5000` is free.

If dashboard opens but values do not update:

1. Check that game session is active and API is reachable.
2. Verify endpoint names in `src/api_endpoints.json` match current TSW API responses.

## Development Notes

- The web server runs in a thread; the poller runs concurrently.
- The app binds to `0.0.0.0:5000` so it can be reached from other devices on your LAN.
